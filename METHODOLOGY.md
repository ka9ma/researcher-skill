# リサーチ手法ガイド

## 適切なユースケース

### 推奨される用途

| 用途 | 説明 |
|------|------|
| **探索的リサーチ** | 本格調査前のペインポイント発見 |
| **仮説生成** | 実ユーザーでテストする仮説の作成 |
| **コンセプトスクリーニング** | 製品アイデアの初期フィルタリング |
| **セグメント探索** | 大規模なセグメント差異の理解 |
| **迅速な反復** | 製品開発中のクイックフィードバック |

### 限界

- 実ユーザーリサーチの**代替ではない**
- シミュレート回答は**学習データのパターンを反映**
- 真に新しい・予期せぬインサイトは得られにくい
- **一次調査を補完**するものとして使用

---

## サンプルサイズガイドライン

| リサーチ目的 | 最小 | 推奨 | 最大有用 |
|-------------|------|------|----------|
| 探索的発見 | 50 | 100-200 | 500 |
| セグメント比較 | 100/セグメント | 200/セグメント | 500/セグメント |
| コンセプトテスト | 200 | 500 | 1000 |
| 統計的検証 | 500 | 1000 | 5000 |

### 注意点
- 大きすぎるサンプルは処理時間が増え、インサイトの質は上がらない
- セグメント比較時は各セグメントに十分なサンプルを確保

## 質問設計

### 効果的な質問の特徴

1. **オープンエンド**だが焦点が明確
2. **具体的な状況**に基づく
3. **感情**に訴える
4. **製品開発**に活用可能

### 良い質問の例

#### ペインポイント発見

| 悪い例 | 良い例 |
|--------|--------|
| 仕事で困っていることは？ | 今日の仕事で一番イライラした出来事を教えてください |
| 何か改善してほしいことは？ | 今週、仕事で最も時間を無駄にしたと感じた瞬間は？ |

#### コンセプト評価

| 悪い例 | 良い例 |
|--------|--------|
| この商品についてどう思いますか？ | もしこの商品が今あなたの目の前にあったら、どう反応しますか？ |
| 使いたいですか？ | このサービスを友人に勧めますか？その理由は？ |

### 質問テンプレート

#### ペインポイント発見用
- 「日々の業務で最もストレスを感じる瞬間は何ですか？」
- 「この職種を選んで最も後悔していることは？」
- 「仕事で一番改善してほしいことは何ですか？」
- 「今週、仕事で最も時間を無駄にしたと感じた瞬間は？」

#### コンセプト評価用
- 「このサービスを初めて知った時、最初に思うことは？」
- 「このアプリを使うとしたら、どんな場面で使いますか？」
- 「このサービスを友人に勧めますか？その理由は？」
- 「このサービスを使わない理由があるとすれば、何ですか？」

## バイアス認識

### データセットの特性

| 項目 | 説明 |
|------|------|
| 人口統計 | 日本の国勢調査データに基づく |
| 職業分布 | 公式労働統計に準拠 |
| 地域分布 | 統計的に調整 |
| ペルソナナラティブ | **LLM生成**（モデルバイアスあり） |

### 主なバイアスリスク

1. **LLMの学習バイアス**: ステレオタイプ的な回答が出やすい
2. **代表性バイアス**: 実際の消費者行動と異なる可能性
3. **確証バイアス**: 期待する回答を過大評価しがち

### 対策

| 対策 | 説明 |
|------|------|
| 方法論の開示 | すべてのレポートで方法論と限界を報告 |
| 実データと照合 | 可能な限り既存の調査データと比較 |
| 仮説生成に限定 | 最終判断には使わない |
| 一次調査で検証 | 重要なインサイトは実ユーザーで確認 |

## 分析フレームワーク

### テーマ別クラスタリング

1. **初期コーディング**: 回答からテーマを特定
2. **軸コーディング**: 関連テーマをカテゴリに分類
3. **選択的コーディング**: コアカテゴリと関係性を特定
4. **定量化**: 出現率と強度指標を算出

### セグメント分析軸

| 軸 | 区分 |
|-----|------|
| 年齢層 | 18-29, 30-39, 40-49, 50-59, 60+ |
| 性別 | 男性, 女性 |
| 地域 | 関東, 近畿, その他 |
| 職業 | 主要職種グループ |
| 趣味 | 関連する趣味クラスター |

## レポート基準

### 必須項目

- [ ] サンプルサイズと構成
- [ ] 方法論の説明
- [ ] 信頼度指標（出現率等）
- [ ] 限界の認識
- [ ] 検証に向けた推奨事項

### レポート構造

```
1. エグゼクティブサマリー（主要発見事項）
2. 方法論（データセット、サンプル、シミュレーション方法）
3. 主要発見事項（テーマ別、出現率、代表的引用）
4. セグメント分析（属性別の傾向）
5. 限界と注意事項
6. 次のステップ（検証推奨事項）
```

## 倫理的考慮

| 原則 | 説明 |
|------|------|
| **補完的使用** | 合成データは人間の声を補完するもの、代替ではない |
| **透明性** | すべてのレポートで方法論を透明に開示 |
| **正直な表現** | シミュレート回答を実ユーザーの引用として提示しない |
| **文化的感受性** | リサーチ設計で文化的感受性を考慮 |
| **合成データの認識** | データの合成的性質を常に認識 |
